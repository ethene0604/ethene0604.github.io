递归

核心思想上来看，算法总体是将大问题分解为若干个子问题进行求解，当子问题足够大，需要递归求解时，我们称之为递归情况，当子问题不需要递归时，称之为基本情况（base case）。 直接或间接地调用自身的算法称为递归算法。用函数自身给出定义的函数称为递归函数。
例如：斐波那契数列的递推公式便体现了递归的递归情况和边界情况。

例题

  1. Ackerman函数 Ackerman函数A(n,m)定义如下： $$ \left\{ \begin{matrix} A(1,0)=2 \\ A(0,m)=1, \ \ \ m>=0\\ A(n,0)=n+2,\ \ \ n>=2\\ A(n,m)=A(A(n-1,m),m-1),\ \ \ n,m>=1 \end{matrix} \right. $$

当一个函数及它的一个变量是由函数自身定义时，称这个函数是双递归函数。

朴实的递归实现，根据函数定义分情况:

int ackerman(int n,int m)
{
	if(m==0&&n==1)
		return 2;

	if(m>=0&&n==0)
		return 1;

	if(m==0&&n>=2)
		return n+2;

	if(m>=1&&n>=1)
		return ackerman(ackerman(n-1,m),m-1);
}
  1. 排列问题
    设计一个递归算法生成n个元素$\{r1 ,r2 ,…,rn\}$的全排列。
    算法思想：固定$r1$，求剩下n-1个元素的全排列，即为：$r1$为第一个元素时的全排列；然后将$r1$和剩下的元素交换，则覆盖了所有情况。
    base-case：当剩下只有一个元素的时候，全排列为它本身
    链接：Permutations - LeetCode
    作业代码实现：
class Solution { 
	public: vector<vector<int>> permute(vector<int>& nums) { 
		int n = nums.size(); 
		if (n == 0) { 
			return {}; 
		} 
		else if (n == 1) { 
			return {{nums[0]}}; 
		} 
		else { 
			vector<vector<int>> result; 
			int num = nums[0]; // 取头部的元素 
			vector<int> rest(nums.begin() + 1, nums.end()); 
			vector<vector<int>> subResult = permute(rest); 
			// 去除头部的一个元素之后的全排列 
			for (const auto& item : subResult) { 
			// 在每个子排列的所有位置和头部元素交换 
				for (int pos = 0; pos <= item.size(); ++pos) { 
					vector<int> tmp = item; 
					tmp.insert(tmp.begin() + pos, num); 
					result.push_back(tmp); 
				} 
			} 
			return result; 
		} 
	} 
};

  1. 整数划分问题 将正整数n表示成一系列正整数之和：$n=n_1+n_2+…+n_k$，
    其中$n_1≥n_2≥…≥n_k≥1，k≥1$。正整数n的这种表示称为正整数n的划分。求正整数n的不同划分个数。 算法思想：设$q(n,m)$为最大加数$n_1$不大于$m$的划分个数，建立递归关系：

    1. $q(n,1)=1$ 即n个1相加
    2. $q(n,m)=q(n,n),m>=n$
      最大加数实际上不能大于n，故有$q(1,m)=1$
    3. $q(n,n)=1+q(n,n-1)$ 将情况分为两种：第一种是给定了最大数为n，只有一种划分，第二种则是递归情况
    4. $q(n,m)=q(n,m-1)+q(n-m,m),n>m>1$ 本质是情况3的特殊到一般，情况分为两种，最大数小于m和最大数给定m
    $$ T(n)=\left\{ \begin{matrix} O(1)\ \ n<4\\ 2T(n/2)+O(n)\ \ n>=4 \end{matrix} \right. $$$$ q(n,m)=\left\{ \begin{matrix} 1\ \ n=1且m=1\\ q(n,n)\ \ n\lt m\\ 1+q(n,n-1)\ \ n=m\\ q(n,m-1)+q(n-m,m)\ \ n>m>1 \end{matrix} \right. $$

代码实现：

int divide(int n, int m)
{
	int res;
	if(n==1||m==1)
		res=1;
	else if (n < m)
		res = divide(n, n);
	else if (n == m)
		res = 1 + divide(n, n - 1);
	else
		res = divide(n, m - 1) + divide(n - m,m);
	return res;
}
注意
递归式的简洁性可能会掩盖它们的低效率

C++/Java没有明确的递归深度。如果超过了最大堆栈大小（在Windows上默认为1/2MB, Linux上可达8MB ）
Java 中 -Xss参数指定最大栈内存，也就是函数调用的最大深度。

分治

用原文可以很好的解释分治策略——divide and conquer，即分而治之。 分治策略分为三个步骤：

  • 分解 自顶向下
  • 解决
  • 合并 自底向上

递归式

上面的Ackerman函数和整数划分问题都整理出了递归式。通过递归式，我们可以很自然的刻画分治算法的运行时间。
求解递归式有三种方法：

  • 代入法
  • 递归树法
  • 主方法(即主定理)

分治法所能解决的问题一般具有几个特征：

  • 该问题的规模缩小到一定的程度就可以容易地解决；
  • 该问题可以分解为若干个规模较小的相同问题;
  • 利用该问题分解出的子问题的解可以合并为该问题的解；
  • 该问题所分解出的各个子问题是相互独立的，即子问题之间不包含公共的子问题。

这条特征涉及到分治法的效率，如果各子问题是不独立的，则分治法要做许多不必要的工作，重复地解公共的子问题，此时虽然也可用分治法，但一般用动态规划较好。

4. 合并排序
分而治之的典型示例，分解到足够小的子问题再由下而上进行合并。
对于两个有序数组的合并：

void Merge(int arr[], int low, int mid, int high) {
	int i = low, j = mid + 1, k = 0; //双指针
	int* temp = new int[high - low + 1];
	while (i <= mid && j <= high) {
		if (arr[i] <= arr[j]) //较小的先存入temp中
			temp[k++] = arr[i++];
		else
			temp[k++] = arr[j++];
	}
	while (i <= mid)//若比较完之后，第一个有序区仍有剩余，则直接复制
		temp[k++] = arr[i++];
	while (j <= high)//同上
		temp[k++] = arr[j++];
	for (i = low, k = 0; i <= high; i++, k++)
		arr[i] = temp[k];
	delete[]temp;
}

void MergeSort(int arr[], int low, int high) {
	if (low >= high) {
		return;
	}
	int mid = low + (high - low) / 2;
	//分治
	MergeSort(arr, low, mid);
	MergeSort(arr, mid + 1, high);
	Merge(arr, low, mid, high);
}

复杂度分析：$T(n)=2T(n/2)+C(n)$
$C(n)$为合并阶段进行两组数据比较的次数，最坏情况下为$n-1$
根据主定理，归并排序的最坏时间复杂度和平均时间复杂度都为$O(nlogn)$
优点：稳定性
缺点：需要额外的线性空间，不能原地排序。 5. 快速排序
原地排序，算法花销主要在于划分节点，而不用再去合并解。
从左到右扫描：从第二个元素开始，遇到第一个大于等于中轴的元素停止 →指针i
从右到左扫描：从最后一个元素开始，遇到第一个小于等于中轴的元素停止 →指针j

为什么当遇到与中轴元素相等的元素时值得停止扫描？
事实上不中止也是可以的，但是当遇到很多相同元素的数组时，用这个方法可以将数组分的更加平均，从而大大提升算法运行速度。

两次扫描全部停止以后，根据扫描的指针是否相交，会发生3种不同的情况：

  • i<j：简单交换A[i],A[j]，接着扫描
  • i>j，把中轴和A[j]交换，得到一个划分。
  • i=j，指向元素的值等于中轴

时间复杂度讨论：
最好：如果所有的分裂点位于相应子数组的中点，这就是最优的情况
$T(n)=2T(n/2)+n$
得到$O(nlogn)$
最坏：所有的分裂点都趋于极端，两个子数组有一个为空，而另一个子数组仅仅比被划分的数组少一个元素。
$T(n)=T(n-1)+n+1$得到$O(n^2)$

线性时间选择

给定线性序集中n个元素和一个整数k，1≤k≤n，要求找出这n个元素中第k小的元素。
最简单的方法不过是先排序再取值，假如采用归并排序，那么得到$T(n)=\Theta(nlogn)$
探究更简单的方法：

参考《算法导论》第九章

线性时间选择以快速排序为模型，不同的是，快速排序会递归处理划分的两边，而这里只需要处理划分的一边，根据主元的变化不断判断目标位于哪一个划分。

template<class Type>
Type RandomizedSelect(Type a[],int p,int r,int k)
{//p和r代表区间始末点，k代表想要的第k个元素
	if (p==r) //始末点相同的情况此时k必然等于1
		return a[p];
	//选取数组中的一个数，进行快速排序，原地将数组分为两部分
	int i=RandomizedPartition(a,p,r),
	j=i-p+1;//选取的pivot的站位
	if (k<=j) //在小的一块
		return RandomizedSelect(a,p,i,k);
	else //在大的一块
		return RandomizedSelect(a,i+1,r,k-j);
}

Select算法

保证数组得到一个好的划分，使得划分更加均匀

  1. 将n个输入元素划分成$\lfloor n/5\rfloor$个组，每组5个元素，至多有一组由剩下的$n mod 5$个元素组成。
  2. 用插入排序，将每组中的元素排好序，取出每组的中位数，共$\lceil n/5\rceil$个。
  3. 调用Select来找出这$\lceil n/5\rceil$个中位数的中位数x。如果$\lceil n/5\rceil$是偶数，就找它的2个中位数中较大的一个。
  4. 按上一步中找到的x作为中轴，对全部 n个元素进行划分，有k-1个数小于x。
  5. 若$i==k$，返回x。若i小于k，在低区调用Select找出第i小的元素，否则在高区调用Select。
Type Select(Type a[], int p, int r, int k)
{
	if (r-p<75) {//用某个简单排序算法对数组a[p:r]排序;
		return a[p+k-1];
	}

	for ( int i = 0; i<=(r-p-4)/5; i++ ){
		//将a[p+5*i]至a[p+5*i+4]的第3小元素与a[p+i]交换位置;
		//找中位数的中位数，r-p-4即上面所说的n-5
		Type x = Select(a, p, p+(r-p-4)/5, (r-p-4)/10);
		int i=Partition(a,p,r, x), j=i-p+1;
		if (k<=j) 
			return Select(a,p,i,k);
		else 
			return Select(a,i+1,r,k-j);
	}
}

最接近点对问题

给定平面上n个点的集合S，找其中的一对点，使得在n个点组成的所有点对中，该点对间的距离最小。

首先从一维问题入手，此时，S中的n个点退化为x轴上的n个实数 x1,x2,…,xn。最接近点对即为这n个实数中相差最小的2个实数。
假设我们用x轴上某个点m将S划分为2个子集S1和S2 ，基于平衡子问题的思想，用S中各点坐标的中位数来作分割点。递归地在S1和S2上找出其最接近点对$\{p1,p2\}$和$\{q1,q2\}$，并设$d=min\{|p1-p2|,|q1-q2|\}$，S中的最接近点对或者是$\{p1,p2\}$，或者是$\{q1,q2\}$，或者是某个$\{p3,q3\}$，其中p3∈S1且q3∈S2。
如果S的最接近点对是$\{p3,q3\}$，即$|p3-q3|m的距离不超过d，即$p3∈(m-d,m]，q3∈(m,m+d]$。
由于在S1中，每个长度为d的半闭区间至多包含一个点(否则必有两点距离小于d)，并且m是S1和S2的分割点，因此$(m-d,m]$中至多包含S中的一个点。
由图可以看出，如果$(m-d,m]$中有S中的点，则此点就是S1中最大点。
因此，我们用线性时间就能找到区间$(m-d,m]$和$(m,m+d]$中所有点，即p3和q3。从而我们用线性时间就可以将S1的解和S2的解合并成为S的解。

循环日赛程表

有$n=2^k$名选手，设计一个满足以下要求的比赛日程表： (1)每个选手必须与其他n-1个选手各赛一次； (2)每个选手一天只能赛一次； (3)循环赛一共进行n-1天。

解法：将日程表设计成 n 行 n 列的表，在表中第 i 行、第 j+1 列处填入第 i 个选手在第 j 天所遇到的对手（第一列是运动员编号）。
把n名选手分成两部分：1～n/2 和 (n/2+1)～n；
1～n/2日程表A：由数字1～n/2组成的(n/2)×(n/2)表格；
(n/2+1)～n日程表B：由数字(n/2+1)～n组成的(n/2)×(n/2)表格；
n名选手日程表是一个n×n的表格，把行和列分别分成两半，可以得到四个(n/2)×(n/2)的表格，记为L1，R1，L2，R2，其中L1与A完全一致，L2与B完全一致；
而R2与L1可以完全一致，因为这一块的选手安排可以照搬到第一部分而不重复，同理R1和L2一致。 递归边界：n=2的时候，只要安排两个人对战即可。

最大字段和

链接：53. 最大子数组和 - 力扣（LeetCode）
给你一个整数数组 nums ，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

子数组是数组中的一个连续部分。

本题可以用动态规划解答，但力扣官方提示可以采用分治解答。 lsum 表示[l,r]内以l为左端点的最大子段和
rsum表示[l,r]内以r为右端点的最大子段和
msum表示[l,r]内的最大子段和 isum表示[l,r]的区间和 简称[l,m]为[l,r]的左子区间，[m+1,r]为[l,r]的右子区间，首先最好维护的是isum，区间[l,r]的isum就等于左子区间的isum加上右子区间的isum。
对于[l,r]的lsum，存在两种可能，它要么等于左子区间的lsum，要么等于左子区间的isum加上右子区间的lsum，二者取大。
对于[l,r]的rsum，同理，它要么等于右子区间的rsum，要么等于右子区间的iSum加上左子区间的rsum，二者取大。
当计算好上面的三个量之后，就很好计算msum了。我们可以考虑[l,r]的msum对应的区间是否跨越m

可能不跨越m，也就是说[l,r]的msum可能是左子区间的msum和右子区间的msum中的一个；它也可能跨越m，可能是左子区间的rsum和右子区间的lsum求和。三者取大。

struct assist{//结构体存储辅助变量
        int lsum;
        int rsum;
        int msum;
        int isum;
    };

class Solution {
public:
    assist getassist(vector<int>& nums,int m,int n){//区间的辅助
        if(m==n){//递归边界
            int i=nums[m];//区间长度为1
            return{i,i,i,i};
        }
        else{
            int mid=(m+n)/2;
            assist left=getassist(nums,m,mid);
            assist right=getassist(nums,mid+1,n);
            return{
                max(left.lsum,left.isum+right.lsum),
                max(right.rsum,right.isum+left.rsum),
                max(left.rsum+right.lsum,(max(left.msum,right.msum))),
                left.isum+right.isum
            };
        }
    }
    int maxSubArray(vector<int>& nums) {
        int n=nums.size();
        int mid=n/2;
        if(n==1){
            return nums[0];
        }
        else{
            return getassist(nums, 0, n-1).msum;
        }
    }
};

链接：1823. 找出游戏的获胜者 - 力扣（LeetCode）
n 个人标号0,1,2…n-1逆时针站一圈，从 0 号开始，每一次从当前的人逆时针数k个，然后让这个人出局。问最后剩下的人是谁。
规律：为了创造递归条件，我们希望，n个人序列杀了1个人之后的序列可以和n-1个人的序列产生联系：

f(10,3):会杀掉2，杀掉后认为3号开头：3-4-5-6-7-8-9-0-1-2
9个人的序列：0-1-2-3-4-5-6-7-8
认为：$f(10,3)=(f(9,3)+3)\%10$ 即$f(n,m)=(f(n-1,m)+m)\%n$

class Solution { 
	public: 
	int findTheWinner(int n, int k) { 
		if (n <= 1) 
			return n; 
		int ans = (findTheWinner(n - 1, k) + k) % n; return ans==0?n:ans; 
	} 
};

递归实现乘法

链接：面试题 08.05. 递归乘法 - 力扣（LeetCode）

用递归的方式实现乘法，不能显式出现乘号：当A为偶数：$A*B=A/2*B+A/2*B$
当A为奇数：$A*B=B+A/2*B+A/2*B$

class Solution {
public:
    int multiply(int A, int B) {
        if (A == 0) {
            return 0;
        } else if (A == 1) {
            return B;
        } else {
            if (A % 2 == 0) {
                int i = multiply(A >> 1, B);
                return i << 1;
            } else {
                return B + (multiply(A >> 1, B) << 1);
            }
        }
    }
};

两个有序数组找第k个数

链接：4. 寻找两个正序数组的中位数 - 力扣（LeetCode）
给定两个大小分别为 m 和 n 的正序（从小到大）数组 nums1 和 nums2。请你找出并返回这两个正序数组的 中位数 。

算法的时间复杂度应该为 O(log (m+n))

课堂上要求用两种解法解答问题：
解法1：通过分解尝试寻找第k个数，由于牵扯到两个数组，所以各选k/2个进行划分，这样第k个数就可能从二数组划分的边界产生。
递归终点：显然，一方为空时，就可以直接求解了；此处假设$mk=1时，即最小的数，我们可以直接比较A和B数组的首个元素即可，也就是$(m,n,1)$时,直接输出二者最小值
合并子问题：从A/B各选第k/2个数进行比较，如下情况

  • Ai=Bi时，二者左边恰好有k个数,那么这两个就是所求解
  • Ai＜Bi时，Ai左边的数肯定不会是问题的解，所以我们直接求解剩下的右半部分即可$(m-k/2,n,k/2)$，问题规模减小。
  • Ai>Bi时类似同理可得
class Solution { 
private: 
	int find(vector<int>& nums1, vector<int>& nums2, 
		int i, int j, int k, int l1, int l2) { 
		if (j == l2) { //数组2为空
			return nums1[i + k - 1]; 
		} 
		if (i == l1) { //数组1为空
			return nums2[j + k - 1]; 
		} 
		if (k == 1) { //求当前区间最小数
			return min(nums1[i], nums2[j]); 
		} 
		int half=k/2; 
		int newIdx1 = min(i + half, l1) - 1, pivot1 = nums1[newIdx1]; 
		int newIdx2 = min(j + half, l2) - 1, pivot2 = nums2[newIdx2]; 
		if (pivot1 <= pivot2) { //抛弃数组1前半部分
			return find(nums1, nums2, newIdx1 + 1, 
				j, k - (newIdx1 - i + 1), l1, l2); 
		} 
		else { 
			return find(nums1, nums2, i, newIdx2 + 1,
				 k - (newIdx2 - j + 1), l1, l2); 
		} 
	} 
public: 
	double findMedianSortedArrays(vector<int>& nums1, vector<int>& nums2) { 
		int len1 = nums1.size(), len2 = nums2.size(); 
		int total = len1 + len2; 
		if (total % 2 == 1) { 
			int mid = total / 2; 
			return find(nums1, nums2, 0, 0, mid + 1, len1, len2); 
		} 
		else { 
			return (find(nums1, nums2, 0, 0, total / 2, len1, len2) + 
			find(nums1, nums2, 0, 0, total / 2 + 1, len1, len2)) / 2.0; 
		} 
	} 
};

解法2：除去找第k个数的方法，我们还可以尝试折半，也就是先不考虑k，而是先折半，再判断。 递归终点：同于解法一，$(0,n,k)$直接输出数组B的第k个即可，此处不需要求解k/2，所以对k没有要求
合并子问题：类似解法一，只不过分别选取了m/2和n/2个数而已。
假设此时的Ai<Bi（相反情况类似），此时的Ai的序数$kA∈[m/2,m/2+n/2]$,Bi的序数$kB∈[m/2+n/2,m+n/2]$。比较k与m/2+n/2的关系：

  • k<m/2+n/2，Bi右边的肯定不满足要求了，所以直接抛弃，求解$(m.n/2,k)$即可
  • k>m/2+n/2，Ai左边的肯定不满足要求了，所以直接抛弃，求解$(m/2,n,k-m/2)$即可
  • 相等情况,可随意抛弃
class Solution { 
private: // l1, l2 是当前段的长度（不是绝对下标边界） 
	int find(vector<int>& nums1, vector<int>& nums2, 
	int i, int j, int k, int l1, int l2) { 
		//剩余区间为空
		if (l1 == 0) 
			return nums2[j + k - 1]; 
		if (l2 == 0) 
			return nums1[i + k - 1]; 
		//剩余区间折半
		int half1 = l1 / 2; 
		int half2 = l2 / 2; 
		int mi = i + half1; 
		int mj = j + half2; 
		int Ai = nums1[mi]; 
		int Bi = nums2[mj]; 
		if (Ai <= Bi) { 
			if (k <= half1 + half2 + 1) { //抛弃B的右半段
				return find(nums1, nums2, i, j, k, l1, half2); 
			} 
			else { //抛弃A的左半段
				return find(nums1, nums2, mi + 1, j, k - (half1 + 1), 
				l1 - half1 - 1, l2); 
			} 
		} 
		else { 
			if (k <= half1 + half2 + 1) { 
				return find(nums1, nums2, i, j, k, half1, l2); 
			} 
			else { 
				return find(nums1, nums2, i, mj + 1, 
				k - (half2 + 1), l1, l2 - half2 - 1); 
			} 
		} 
	} 
public: 
	double findMedianSortedArrays(vector<int>& nums1, vector<int>& nums2) { 
		int l1 = nums1.size(), l2 = nums2.size(); 
		int total = l1 + l2; 
		if (total % 2 == 1) { 
			return find(nums1, nums2, 0, 0, total / 2 + 1, l1, l2); 
		} 
		else { 
			return (find(nums1, nums2, 0, 0, total / 2, l1, l2) +
			 find(nums1, nums2, 0, 0, total / 2 + 1, l1, l2)) / 2.0; 
		} 
	} 
};
